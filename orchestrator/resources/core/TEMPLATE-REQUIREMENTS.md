## TEMPLATE REQUIREMENTS INSTRUCTIONS

### General
- **Replace all curly-brace placeholders** with actual project data using format `{PLACEHOLDER}`
- **Follow standardized placeholder format** using appropriate convention
- **Include target identifier** and selected agent/method/approach
- **Use consistent formatting** and structure throughout all templates
- **Provide clear rationale** for all decisions and selections made
- **ONLY USE** template structure for reporting when instructed to use template
- **STRICTLY FORBIDDEN** to use emojis when writing or modifying files
- **Remove all markdown comments** from resulting documents after template use, EXCEPT `<!-- Updated … -->` timestamp comments which MUST be preserved

### Adaptive Guidelines
- **OMIT sections and bullet-points** when not applicable
- **ADD bullet-points** that are not mentioned in the base template
- **NEVER** add sections, only extend bullet-points in existing sections

### Context
- **Include hierarchical context** where applicable
- **Show scope assessment** and complexity analysis when relevant
- **Detect integration requirements** to understand project fit
- **Parse development progress files** for accurate project status
- **Scan dependencies** for accurate tech stack
- **Use git commands** to determine current branch and recent progress
- **Always show detected technologies** with versions and descriptions
- **Count test files** and analyze test configuration
- **Extract run scripts** and key development commands
- **Identify key architectural patterns** from codebase structure
- **Determine current status progression** by analyzing project progression system

### Quality
- **Use precise identifiers** with actual values, not placeholders in final output
- **Include working links** for cross-references
- **Maintain audit trail** with return codes and status transitions
- **Mention alternatives** considered with rationale for final selection

### Formatting
- **Use markdown headers** with consistent hierarchy
- **Use bullet points** for requirements, numbered lists for sequences
- **Use triple backticks** with language identifiers
- **Use markdown link format** with wrapping pattern `[…](<…>)` to support spaces in URLs
- **NEVER** escape URLs in markdown links
- **Use CONST_CASE** for states and return codes
