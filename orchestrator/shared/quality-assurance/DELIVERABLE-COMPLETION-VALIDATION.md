## DELIVERABLE COMPLETION VALIDATION
- Before returning success status, verify ALL deliverables are complete

### **CRITICAL REQUIREMENTS**
- All functional requirements fully implemented and working
- Zero placeholders, stubs, or partial implementations
- Complete test coverage for all critical paths

### Functional
- **Verify requirements** ensuring all specifications are met
- **Test user flows** validating end-to-end functionality
- **Check edge cases** testing boundary conditions and error paths
- **Validate integration** ensuring component interactions work correctly
- **Confirm completeness** verifying no missing features or capabilities

### Technical
- **Review code quality** checking adherence to project standards
- **Validate architecture** ensuring proper design patterns are followed
- **Test performance** verifying response times and resource usage
- **Check security** validating input sanitization and access controls
- **Verify compatibility** testing across required platforms and environments

### Quality
- **Execute test suites** running all automated tests
- **Perform regression** ensuring no existing functionality is broken
- **Validate documentation** checking technical accuracy and completeness
- **Review error handling** testing failure scenarios and recovery
- **Confirm accessibility** validating WCAG compliance where applicable
