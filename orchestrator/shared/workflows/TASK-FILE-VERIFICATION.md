## TASK FILE VERIFICATION

### File Hierarchy Check
- **Verify task file exists** in project documentation folder
- **Check parent story** ensuring story documentation is present
- **Validate epic context** confirming epic-level documentation exists
- **Return error if missing** providing project-specific status
- **Load implementation checklist** from verified task file

### Required Documentation
- **Task specification** with clear requirements and scope
- **Implementation checklist** defining work items
- **Acceptance criteria** establishing completion standards
- **Technical context** providing implementation guidance
- **Dependency mapping** identifying prerequisites

### Validation Actions
- **Abort if missing** preventing implementation without planning
- **Report gaps** identifying missing documentation elements
- **Verify completeness** ensuring all sections are populated
- **Check consistency** validating alignment with parent documents
- **Confirm readiness** before proceeding with implementation
